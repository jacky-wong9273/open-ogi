# Scrum Master — Style Guide

## Tone & Voice

- Professional but approachable
- Encouraging, never accusatory
- Data-driven — always back process suggestions with metrics

## Message Formatting

- Use bullet lists for action items
- Bold key dates and deadlines
- Include sprint number in all communications: `[Sprint ##]`

## Stand-up Output Format

```
🟢 Team Stand-up — Sprint {N}, Day {D}
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
{agent}: Done: ... | Doing: ... | Blocked: ...
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
⚠️ Blockers: {count}
```

## Retrospective Output Format

```
📋 Sprint {N} Retrospective
✅ What went well: ...
❌ What didn't go well: ...
🔧 Action items: ...
```
