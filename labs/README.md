# Labs — Sample Scrum Team

This directory contains pre-built abstract agents, skills, and tools for a
software-engineering Scrum team. Load them into the Labs server to bootstrap
your first agentic workflow.

## Agents

| Agent         | Role                                                       |
| ------------- | ---------------------------------------------------------- |
| scrum-master  | Sprint facilitation, blocker removal, process enforcement  |
| tech-lead     | Architecture decisions, code reviews, technical mentorship |
| developer     | Feature implementation, unit testing, PR authoring         |
| qa-engineer   | Test planning, test execution, bug reporting               |
| product-owner | Backlog grooming, acceptance criteria, stakeholder comms   |

## Skills

| Skill           | Purpose                                      |
| --------------- | -------------------------------------------- |
| code-review     | Structured code review with checklist output |
| sprint-planning | Story estimation, sprint goal definition     |
| daily-standup   | Status collection and blocker identification |
| retrospective   | Gather feedback, identify improvements       |
| bug-triage      | Classify, prioritise, assign incoming bugs   |

## Tools

| Tool               | Purpose                                             |
| ------------------ | --------------------------------------------------- |
| telegram-messenger | Send / receive messages via Telegram Bot API        |
| whatsapp-messenger | Send / receive messages via WhatsApp Cloud API      |
| jira-integration   | Create/update Jira issues, query boards             |
| git-operations     | Clone, branch, commit, push via local git CLI       |
| code-analyzer      | Static analysis / linting via ESLint + custom rules |
